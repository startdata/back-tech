## 구성

- 자원 URI
- 행위 HTTP METHOD
- 표현

## 가이드

- 첫 번째, URI는 정보의 자원을 표현
- 두 번째, 행위는 HTTP Method(GET, POST, PUT, DELETE)로 표현
- 행위가 들어가는 부분은 URI에 넣지 않는다. (ex.show,insert)

## 주의사항

- 슬래시는 계층관계를 나타내는데 사용
- 마지막 문자는 슬래시를 쓰지않는다.
- 긴경우에는 하이픈
- 소문자 사용
- 파일확장자는 URI에 포함하지 않는다.

### 리소스간의 관계를 표현

```
/리소스명/리소스ID/관계있는리소스
GET user/{userid}/device
```
